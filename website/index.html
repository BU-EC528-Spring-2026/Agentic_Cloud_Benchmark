Overview of ACBench

ACBench focuses on agentic cloud work where the solution is rarely just a single code edit. Scenarios combine a fixed repository state, bounded evidence, explicit success criteria, and reproducible execution paths.

The benchmark includes small local fixtures for pipeline validation and GitHub-derived OpenClaw reproductions for more realistic failures. GitHub scenarios are localized into stable fixtures rather than depending on a moving upstream branch.

Task Categories

ACBench separates evaluation into code, ops, and combined capabilities while preserving a shared scenario schema.

</>

Code Repair

Agents receive a buggy local repository, task context, and test commands. They must generate a patch that fixes fail-to-pass tests without breaking pass-to-pass behavior.

Input
Buggy fixture repository and issue context
Output
Patch in diff format
Evaluation
Build, tests, and repair criteria
⌕

Ops Incident Analysis

Agents analyze incident evidence such as logs, symptoms, reproduction steps, and service context. Responses are graded against detection, localization, and remediation rubrics.

Input
Incident description and visible evidence
Output
Structured diagnostic assessment
Evaluation
Rubric-based incident scoring
⇄

Combined Workflows

Agents must first reason through operational symptoms, then apply code-level fixes that resolve the underlying failure and validate the repaired service.

Input
Incident plus repository fixture
Output
Diagnosis and repair patch
Evaluation
Ops rubric plus code tests

Key Results

The project is ready to run local and GitHub-derived benchmark suites through gold patches, OpenAI agents, Anthropic agents, or custom agent profiles.

Reproducible Fixtures

Every scenario pins its source and execution path so evaluations avoid branch drift and hidden remote-state changes.

Provider-neutral Agent Profiles

OpenAI and Anthropic integrations are built in, while custom agents can be loaded through module references or JSON profiles.

Multi-signal Scoring

Code outcomes capture build and test transitions; ops outcomes capture detection, localization, and repair-quality evidence.
